# Stage 1: 外部数据准备

你现在负责“阶段1：外部数据准备”。

先阅读当前 repo、配置与数据目录，再开始修改。
只做当前阶段，不要实现 LLM 抽取、图构建、训练或实验。

## 已知条件

- FNSPID 原始数据已经存在，不要重新下载；
- 其余外部数据都需要从公开来源下载或整理；
- 项目评估区间是 2016-01-01 到 2023-12-31；
- 数据准备起点是 2015-09-01。

## 本阶段目标

1. 下载或整理 SP500 股票池名单；
2. 下载 2015-09-01 到 2023-12-31 的股票价格数据；
3. 下载或整理 SP500 市场指数序列，用于 feat_mkt_ret_1d；
4. 生成最初版公司名/ticker/alias 对照表；
5. 将所有原始输入统一落到 data/raw/ 下；
6. 写清楚数据来源、版本和限制。

## 推荐实现主线

- **SP500 股票池**：优先做“固定研究股票池”模式，先从公开来源抓取成一份冻结列表，并在文档中明确这是固定 universe 版本；
- **价格数据**：优先使用 yfinance 或你认为最可靠的公开来源，必须下载 adj_close 用于收益计算，并在文档中记录数据来源和潜在限制；
- **市场特征**：优先下载 SP500 指数收益；若数据源不便，可用明确记录的替代方案，但必须在文档里写清楚；
- **alias seed**：基于股票池中的 company_name 生成最初版别名字典。

## 必须创建或补全

- src/data/download_prices.py
- src/data/download_universe.py
- src/mapping/alias_table.py（先生成初版）
- configs/data/universe.yaml
- scripts/run_download_data.sh（如需要）
- data/raw/prices/
- data/raw/universe/
- data/raw/mapping/

## 要求

- 下载范围必须从 2015-09-01 开始；
- 所有脚本要支持断点续跑；
- 每个下载产物都要带 source/version/created_at 元数据或旁路说明文件；
- 若某些 ticker 下载失败，不要中断整个流程，要记录失败列表；
- 暂时不要做复杂的历史成分股动态切换，先落地固定 universe 版本；
- 在 README 或 docs 中明确写出这一限制。

## 修改后运行 smoke test

- 至少抽样验证几只股票的数据可读取；
- 验证市场指数数据存在；
- 验证 universe 文件存在并字段完整；
- 验证 alias seed 文件生成成功。
